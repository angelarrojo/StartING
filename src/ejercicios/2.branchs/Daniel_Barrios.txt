Ejercicio 2

2.1. Preguntas

¿Qué es un branch?
-Un brach es una rama donde podemos modificar la información sin afectar al proyecto final.

¿Por qué pueden ser útiles los branches?
-Las ramas pueden ser útiles para que varias personas trabajen a la vez en un mismo y cada uno este en su propia rama.

¿Cómo se crea una branch?
-git branch <nombreRama>

¿Cómo se cambia a una branch?
-git checkout <nombreRama>

¿Cómo se elimina una branch?
-git branch -d <nombreRama>

¿Cómo se crea una branch y se cambia a ella en un solo paso?
-git checkout –b <nombreRama>

¿Qué es un merge?
-Un merge es cuando se unen dos ramas para juntar esa información.

¿Cómo se realiza un merge?
-git merge <nombreRama>

¿Qué es un tag?
-Un tag es una etiqueta en un commit para señalar algo muy importante.

¿Cómo se crea un tag?
-git tag -a <lo que queramos decir> -m "mensaje"
