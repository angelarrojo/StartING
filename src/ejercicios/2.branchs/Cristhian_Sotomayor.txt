1. ¿Qué es un branch?
Un branch es un conjunto llamado commits las ramas depende mucho de la rama master que eso se genera con el primer commit
2. ¿Por qué pueden ser útiles los branches?
Los branches pueden ser utiles porque te permite trabajar en nuevas funciones de forma aislada se puede corregir errores sin afectar a la rama principal
3. ¿Cómo se crea una branch?
Para poder crear un branch debemos poner git branch y el nombre de la rama
4. ¿Cómo se cambia a una branch?
Para poder cambiarlo debemos poner git checkout y nombre de la rama
5. ¿Cómo se elimina una branch?
Para poder eleminar un branch debemo poner git branch -d y el nombre de la rama
6. ¿Cómo se crea una branch y se cambia a ella en un solo paso?
Debemos poner git checkout -b y el nombre de la rama
7. ¿Qué es un merge?
Un merge es para poder juntar los cambios de una rama a otra
8. ¿Cómo se realiza un merge?
Para poder realizar un merge primero hay que estar en la rama donde queremos hacer los cambios y despue ponemos git merge y depsue el nombre de la rama y el origuen
9. ¿Que es un tag?
Un tag es como una etiqueta que se utiliza para poder marcar una version
10. ¿Cómo se crea un tag?
Para poder crear un tag debemos poner git tag -a el nombre del tag -m mensaje
