### 2.1. Preguntas

1. ¿Qué es un branch?
- Un branch es una rama dentro de un repositorio donde se pueden realizar cambios sin afectar la rama main.
2. ¿Por qué pueden ser útiles los branches?
- Los branches son útiles porque permiten trabajar nuevas funcionalidades sin modificar directamente el código principal.
3. ¿Cómo se crea una branch?
- Con el comando "git branch"
4. ¿Cómo se cambia a una branch?
- "git checkout"
5. ¿Cómo se elimina una branch?
- "git branch -d"
6. ¿Cómo se crea una branch y se cambia a ella en un solo paso?
- "git checkout -b nombre-rama"
7. ¿Qué es un merge?
- Un merge es la acción de unir los cambios de una branch dentro de otra branch.
8. ¿Cómo se realiza un merge?
- Un merge se realiza cambiando primero a la rama destino y ejecutando "git merge"
9. ¿Que es un tag?
- Un tag es una etiqueta que se usa para marcar versiones importantes de un proyecto, como una release
10. ¿Cómo se crea un tag?
- git tag nombre-tag
