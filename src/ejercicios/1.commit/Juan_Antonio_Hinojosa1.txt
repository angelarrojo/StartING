1. ¿Cómo se inicializa un repositorio local? (que comando se debe ejecutar?)
Se ejecuta el comando git init y se podra crear un preyecto (repositorio) donde podremos ejecutar y modificar cambios.

2. ¿Cómo hago para que un directorio deje de ser controlado por git? (que comando se debe ejecutar?)
Tienes que poner rm -rf .git para que se elimine el historial que tiene el proyecto.
Esto hace que Git ya no pueda ver lo que haces.

3. Si agrego un archivo a un directorio que ya está siendo controlado por git, ¿está siendo controlado por git?
No, porque cuando ejecutas el comando el archivo siguira sin el segimiento de Git. Hasta que lo pongas.

4. ¿Qué comando se utiliza para agregar un archivo al repositorio local?
Para agregar un archivo necesitanmos primero poner el archivo en el area de preparcion de guardado para luego hacer un commit
 1º- es git add para poder guardar los archivos que tendras que poner antes del guardarlo en un commit
 2º - Pones un commit para guardar los cambios y ponerlos los en el repositorio local.
- git add <archivo>
- git commit -m <descripcion del commit>

5. ¿Cómo determino que archivos fueron modificados? (que comando se debe ejecutar?)
Para determinar el archivo tienes que poner git status que para que te muestre con colores el estado del codigo.

6. ¿Qué comando se utiliza para hacer un commit?
El comando para hacer un commit es git commit -m "mensaje" este comando te permite guardar los cambios en un commit y
 darles una descripcion para que puedas ver que cambios has hecho en el codigo con la frase que has puesto "".

7. En sus propias palabras, ¿qué es un commit?
Un commit es para guardar el cambios que has hecho con el codigo y darle un mensaje para saber que accion o que cambios has hecho anteriormente .
