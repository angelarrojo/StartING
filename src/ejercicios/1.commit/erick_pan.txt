1. ¿Cómo se inicializa un repositorio local? (que comando se debe ejecutar?)
Para iniciar un repositorio local en GIT necesitamos entrar en GIT BASH utilizando el comando cd para cambiar a la capeta y al proyecto, luego para poder ejecutarlo un repositorio local necesitamos poner este comando: git init, significa que hemos creado un repositorio vacío
2. ¿Cómo hago para que un directorio deje de ser controlado por git? (que comando se debe ejecutar?)
Para hacerlo necesitamos poner este comando: git rm -r --cache (rm: eliminar un archivo/directorio) (-r: recursivo) (--cached: elimina los archivos de índice)
3. Si agrego un archivo a un directorio que ya está siendo controlado por git, ¿está siendo controlado por git?
No, porque cuando añades un archivo nuevo te lo clasifica como untrack (sin rastear)
4. ¿Qué comando se utiliza para agregar un archivo al repositorio local?
Para agregar un archivo tienes que poner este comando: git add <nombre_de_archivo>
5. ¿Cómo determino que archivos fueron modificados? (que comando se debe ejecutar?)
Utilizamos: git status, que es un resumen rápido que te dice qué archivo fueron modificado borrado o nuevos
6. ¿Qué comando se utiliza para hacer un commit?
Utilizamos: git commit, para crear un commit pero podemos añadir -m para añadir un mensaje
7. En sus propias palabras, ¿qué es un commit?
Significa que vas a guardar un cambio que has hecho (importante) en el proyecto y que va acompañado de un mensaje
